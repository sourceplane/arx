## 1. Objective

Ship the **C1 resolver** for the orun-component-catalog spec: turn a
workspace path into a fully-populated `sourcectx.WorkspaceState`
deterministically, with injected adapters so it is fully testable
without a real git repo on disk.

Concretely, land:

1. A `Git` adapter interface (HEAD revision, tree hash, branch, ref, tag,
   PR-number-from-CI-env hooks) plus one default implementation. You
   choose between **(a) `go-git` in-process** or **(b) `os/exec` shell-out
   to `git`**. Document the trade-off in the PR body (binary size vs
   subprocess cost vs CI portability) — that paragraph is part of
   acceptance.
2. A `Clock` interface (one method, `Now() time.Time`) and a `Filesystem`
   interface (`Walk`, `Stat`, `ReadFile` — minimum surface needed to
   enumerate catalog-relevant dirty files).
3. The top-level entry point:

   ```go
   func ResolveSourceSnapshot(ctx context.Context, opts ResolveOptions) (WorkspaceState, error)
   ```

   `ResolveOptions` carries (at minimum) `WorkspacePath string`, the
   three adapters (with sensible defaults wired by a `New(...)` /
   `WithDefaults()` helper), the `IntentCatalog` block needed by the
   `CatalogInputHash` (sourced by callers — do NOT couple to
   `intent.yaml` parsing here; accept the canonical bytes via
   `CatalogInputHashInputs`), and CI-event injection hooks (PR number,
   tag, ci-event scope override).
4. Population logic:
   - `HeadRevision` ← short SHA (12+ hex chars) from the Git adapter.
   - `TreeHash` ← short tree SHA (7+ hex chars) from `git rev-parse
     HEAD^{tree}`.
   - `Branch` / `Ref` / `Tag` / `RemoteURL` / `Repo` from the Git
     adapter.
   - `Dirty` + `DirtyHash` ← walk the workspace, filter to the
     **catalog-relevant** file set per `identity-and-keys.md` §7
     (`intent.yaml`, every `component.yaml`, `stack.yaml` /
     `composition.yaml`, gated `package.json` / `Dockerfile` /
     `Chart.yaml` / `*.tf` / `README.md` per `intent.yaml`
     `catalog.inference.*` flags), diff each against `treeHash`, and
     pass the resulting `[]DirtyFile` to the existing `DirtyHash`
     helper.
   - `CatalogInputHash` ← exposed via either a method on
     `WorkspaceState` or a free function over
     `CatalogInputHashInputs` (already in `hash.go`); resolver fills
     `TreeHash` + `DirtyHash` + the resolver-version block.
   - Scope detection follows `WorkspaceState.Scope()` already in
     `model.go`; resolver only fills inputs and lets `Scope()`
     classify.
5. **Error envelope on `--from-ci` no-match** — when caller injects a
   CI event and no matching ref/PR is found, return the same shaped
   error envelope Phase 1 §11 documents (typed sentinel + structured
   reason). Mirror `internal/triggerctx`'s pattern; do not invent new
   error machinery.

## 4. Tests (acceptance is enforced via `make test-state-redesign`)

Required tests, all under `internal/sourcectx/`:

1. **Fixture matrix** — for each spec scope (`branch-main`,
   `branch-<name>`, `pr<num>`, `tag-<name>`, `local-dirty`,
   `local-nogit`, `ci-event`), build a fake adapter set, run
   `ResolveSourceSnapshot`, and assert:
   - `WorkspaceState.Scope()` returns the right enum string.
   - `BuildSourceSnapshotKey(state)` matches the documented example
     from `identity-and-keys.md` §2 byte-for-byte (or a regex when
     hex segments depend on fixture content).
2. **T-IDK-3** (existing in catalogmodel as a key-shape test; here
   exercise the higher-level path) — for one workspace fixture, vary
   the `[]DirtyFile` ordering 1 000 times and assert
   `DirtyHash` + the resolved `SourceSnapshotKey` are byte-identical
   across all orderings.
3. **T-IDK-4** — given a workspace with a known catalog-relevant file
   set, add a non-catalog-relevant file (e.g. `notes.txt`,
   `vendor/foo.go`) to the dirty enumeration input and assert
   `dirtyHash` is **unchanged**. This is the property C1 owes the
   spec.
4. **`--from-ci` no-match error envelope** — inject a CI event that
   doesn't match any ref/PR; assert the typed sentinel + structured
   reason match the Phase 1 §11 shape.
5. **Adapter contract tests** — small unit tests for the default Git
   adapter against a tiny on-disk repo built in `t.TempDir()`. Skip
   on CI if `git` is not available **only if you went the shell-out
   route**; if you went `go-git`, no skip.
