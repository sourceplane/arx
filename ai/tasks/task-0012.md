## Integration Notes
- `revision.UpdateLatestExecutionSummary` lives at
  `internal/revision/manifest.go` (M3 PR-B). Read its godoc for the CAS +
  idempotent-short-circuit semantics before wiring `CreateExecution`. The
  short-circuit fires when the supplied summary is byte-identical to the
  current latest — useful for `MarkTerminal` re-invocation after a crash.
- `revision.WriteRevision` produced the `latest.json` alias via
  `store.Write` (last-write-wins) per Task 0011 verifier adjudication.
  `executionstate` should follow the same convention for any
  `executions/.../latest.json` alias unless `data-model.md` §5 explicitly
  requires `CreateIfAbsent`.
- The M3 resolver uses a seven-branch ladder; the M4 resolver branches are
  a strict subset (no plan-hash branch). Keep the ordering and the
  `ErrAmbiguousArg` discipline aligned so M5 can compose them without
  surprise.
- `data-model.md` §9 event stream: append-only newline-delimited JSON. Reuse
  whatever event-emit helper M3 produced; if M3 inlined event writes,
  document that and inline here too — do not refactor M3 in this PR.
