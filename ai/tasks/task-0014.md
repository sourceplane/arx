## Objective
Close Milestone M4 by shipping `internal/executionstate/bridge.go`:
`Bridge{Store, LegacyRoot, MirrorMode}` plus `MirrorRunnerOutput(ctx, execKey,
revKey, legacyExecID)` with **hardlink-with-copy fallback**, EXDEV detection,
and the `bridge-mirror-failed` event emission per `data-model.md` §9 and the
M4 "Done when" checklist. Wire one synthetic call site in tests so the bridge
is exercised end-to-end without touching production runner code.

## PR Boundary
**In scope:**
1. `internal/executionstate/bridge.go` — new file. Public surface:
   - `type MirrorMode int` (constants: `MirrorModeHardlink`, `MirrorModeCopy`,
     and — if the spec wording supports it — `MirrorModeAuto` defaulting to
     hardlink-with-copy-fallback).
   - `type Bridge struct { Store statestore.StateStore; LegacyRoot string;
     MirrorMode MirrorMode; Clock clock.Clock /* same shim as M3 */ }`.
   - `func (b *Bridge) MirrorRunnerOutput(ctx context.Context, execKey,
     revKey, legacyExecID string) error`. Failures emit a
     `bridge-mirror-failed` event under the execution's event log and return
     `nil` per `implementation-plan.md` §M4 ("Failures emit
     `bridge-mirror-failed` event and return nil"). Hard precondition
     violations (empty key, missing legacyExecID, etc.) still return an error
     wrapping `statestore.ErrInvalid`.
2. `internal/executionstate/bridge_test.go` (and any helper test file you
   need):
   - **Hardlink success path** against a temp dir on a single FS.
   - **Cross-device → copy fallback** via forced `EXDEV` injection. Use a
     small filesystem seam (a `linker` interface or a function variable)
     wrapping `os.Link`; tests inject a stub returning `&os.LinkError{Err:
     syscall.EXDEV}` (or an equivalent platform-portable shape) so the
     fallback path is reachable on macOS/Linux CI without privileged mounts.
   - **`bridge-mirror-failed` event emitted** on both copy-fallback failure
     and on a non-EXDEV link error; assert the event payload via
     `internal/testfx/statefs` helpers.
   - **Idempotent re-mirror** — calling `MirrorRunnerOutput` twice with the
     same inputs must not double-emit and must converge.
3. `Makefile` — keep `internal/executionstate` ≥ 90 % gate; PR-B should
   organically lift coverage above the floor.
4. *Optional, only if needed for testability:* additive helpers in
   `internal/statestore/paths.go` for legacy mirror paths. No new path
   literals in `executionstate/`.

- `internal/runner` integration. M5 owns CLI rewire and runner wiring.

## Integration Notes
- `MirrorRunnerOutput` is invoked by tests only in this PR. Production
  wiring is M5 (`orun run` will call it once per terminal-state transition).
  The bridge MUST tolerate calls against execution rows that already exist
  (idempotent) and against rows that may have been finalized concurrently.
- Per `design.md` §11 risk row "Bridge mirror desync silently corrupts `orun
  status`": bridge failures emit a structured event AND the resolver (PR-A)
  already prefers revision-first with legacy fallback. Keep that contract
  intact — do not change resolver semantics in this PR.
- `data-model.md` §9 event stream is append-only newline-delimited JSON —
  reuse PR-A's event-emit helper (or whatever pattern PR-A inlined). If PR-A
  inlined event writes, document that and inline here too. Do not refactor
  PR-A.
- The R-002 ExecID-shape risk is closed: `SanitizeExecID` round-trips
  GHA-shape `gh-{run_id}-{attempt}-{sha}` identity (verified in Task 0013
  spot-check). PR-B's `legacyExecID` parameter must therefore accept that
  shape without further sanitization in the bridge layer.
