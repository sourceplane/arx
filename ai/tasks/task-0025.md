## Objective

Stand up `internal/catalogresolve` with the first three resolution-pipeline
stages — workspace discovery of `component.yaml` files, schema-validated
load into a typed model, and authored-only inheritance from `intent.yaml`
catalog defaults (lower-precedence layers only). The output of this PR is
a deterministic, in-memory `[]AuthoredManifest` (resolved-but-not-inferred,
not-yet-cross-referenced) consumable by the C2 second PR
(infer + deps + validate + hash). No FS writes, no CLI changes.

## PR Boundary

In scope (one PR):
1. New package `internal/catalogresolve/` with files limited to:
   - `discover.go` — workspace walk + `intent.yaml` `catalog.discovery.exclude`
     filtering; both `component.yaml` and `component.yml`; rejects mixed-form
     in same dir.
   - `load.go` — YAML decode + JSON-Schema validation against the embedded
     `internal/catalogmodel/schema/component-yaml.schema.json`. Returns typed
     `AuthoredComponent` plus `(file, JSONPointer)` provenance trail.
   - `inherit.go` — apply `intent.yaml` `catalog.*` defaults under the
     authored layer, per `resolution-pipeline.md` §3 (map-key-wise for
     labels/annotations; list-wholesale for tags/lists; explicit `[]`
     treated as set).
   - Internal types (`AuthoredManifest`, `Provenance`, `DiscoveryResult`,
     etc.) and a top-level `DiscoverAndLoad(ctx, opts) (DiscoveryResult, error)`
     entry point — explicitly **not** the full `Resolve(ctx, opts)`.
   - Tests + `testdata/` fixtures covering: clean repo, exclude rules,
     `.yml` vs `.yaml` collision, schema-invalid manifest, intent-defaults
     fill, explicit-key precedence, empty-list-vs-unset, missing intent
     catalog block.
2. Wire the new package's coverage gate into the Makefile target the
   verifier hardened in PR #168 (`internal/catalogresolve` ≥ 90 %).
3. No edits to `internal/catalogmodel/` or `internal/sourcectx/` source
   files. Test-only fixture additions there are forbidden.
4. No edits to `intent.yaml`, `examples/intent.yaml`, or any CLI entry point.

Explicit non-goals (defer to Task 0026 / C2 second PR):

## Constraints

1. No raw filesystem path concatenation — use `filepath` helpers per the
   spec's path-conventions rule.
2. Determinism: directory walk uses `filepath.WalkDir` with stable lexical
   sort; result lists are sorted before return.
3. JSON-Schema validation runs against the embedded artifact only — do not
   re-derive the schema from Go types at runtime.
4. Inheritance helpers must keep canonical-encoder-friendly output (no
   `interface{}` round-trips that lose key order; use the catalogmodel
   types directly where possible).
5. Phase 1 invariants preserved: zero edits to `internal/statestore`,
   `internal/revision`, `internal/executionstate`, `internal/triggerctx`.
6. Leaf-clean: `internal/catalogresolve` may import `internal/catalogmodel`
   (and stdlib + go-yaml + the JSON-Schema validator already pulled in by
   C0 if any). It must NOT import `internal/sourcectx`,
   `internal/catalogstore`, or any Phase 1 internal package.
7. Coverage gate is wired into the same Makefile target the C0/C1 gates
   use, mirroring the verifier-attached pattern from PR #168.
