## PR Boundary

In scope (one PR):

1. `internal/statestore/refs.go` — typed ref structs and reader/writer helpers covering all four ref doc shapes in `data-model.md` §6:
   - `LatestRevisionRef`            → `refs/latest-revision.json`     (§6.1)
   - `LatestExecutionRef`           → `refs/latest-execution.json`    (§6.2)
   - `TriggerRef`                   → `refs/triggers/<name>/latest.json` and `refs/triggers/<name>/<scope>.json` (§6.3)
   - `NamedRef`                     → `refs/named/<name>.json`        (§6.4 — format reserved; helpers shipped, no CLI sugar)
   Reader helpers (`ReadLatestRevisionRef`, `ReadLatestExecutionRef`, `ReadTriggerRef`, `ReadNamedRef`) return `(value, ObjectMeta, error)` and propagate `ErrNotFound` through unchanged.
   Writer helpers (`WriteLatestRevisionRef`, `WriteLatestExecutionRef`, `WriteTriggerRef`, `WriteNamedRef`) marshal with deterministic, stable JSON (sorted keys, `\n` terminator, no HTML escaping), then call `Write`. They MUST NOT swallow errors. Writers MAY accept an optional `*ObjectMeta` for CAS-driven updates via a paired `CompareAndSwap`-flavored helper (e.g. `CASLatestRevisionRef(ctx, store, prev *ObjectMeta, next LatestRevisionRef)`); pick the simplest shape that lets M3 do "loser-retries" cleanly.
2. `internal/statestore/indexes.go` — typed index writer plus the `RebuildIndexes()` stub:
   - `RevisionIndexEntry`  → `indexes/revisions/<revisionKey>.json`  (§7.1)
   - `ExecutionIndexEntry` → `indexes/executions/<executionKey>.json`(§7.2)
   - `WriteRevisionIndex(ctx, store, RevisionIndexEntry)` and `WriteExecutionIndex(ctx, store, ExecutionIndexEntry)` use `CreateIfAbsent` (an index entry is immutable once written; rebuild path goes through delete-then-create or a dedicated rebuild helper).
   - `RebuildIndexes(ctx, store) error` — stub that returns `fmt.Errorf("%w: rebuild deferred to M3+", ErrInvalid)`. The stub exists so M3 wiring can compile against the final symbol; real implementation lands when revisions actually populate.
3. Tests:
   - `internal/statestore/refs_test.go` and `internal/statestore/indexes_test.go` exercising round-trip (write→read→equal), missing-ref → `ErrNotFound`, double-write semantics for indexes (second `WriteRevisionIndex` for the same key returns `ErrExists`), CAS race on `LatestRevisionRef` (two writers, exactly one wins), and JSON byte-for-byte stability against fixtures captured via `internal/testfx/statefs.AssertJSONFile`.
   - Path-helper coverage check: every ref/index helper builds its target path through the existing `paths.go` helpers (no string concatenation in `refs.go` / `indexes.go`).
4. `make test-state-redesign` continues to enforce the ≥ 95 % coverage gate on `internal/statestore`. PR-C should leave the package at **≥ 96 %** (the M2 stretch target) — the new files are mostly-thin marshal wrappers and should be easy to cover.

## Constraints

1. JSON marshal MUST be deterministic and stable across Go versions: use `encoding/json` with `MarshalIndent("", "  ")` (or a small custom canonical encoder if `MarshalIndent` doesn't sort keys deterministically for `map` fields — these structs only use named fields, so the standard encoder is fine), trailing newline, no HTML escaping. Round-trip tests MUST compare bytes, not just decoded values.
2. Every ref/index file path MUST go through the helpers in `paths.go`. Zero string concatenation for paths in the new files.
3. Reader helpers MUST return the four sentinels via `errors.Is` only. Do NOT introduce any string-sniffing fallback.
4. CAS helpers MUST use `*ObjectMeta` from the prior read (`Revision` field) — they MUST NOT re-read inside the helper. Loser-retries are the caller's responsibility per `state-store.md` §6.
5. Index writers MUST use `CreateIfAbsent`. Re-writing an existing index entry returns `ErrExists`. Idempotency is the caller's job (M3 will dedupe before write).
6. Package stays leaf-clean: `internal/statestore` MUST NOT import any other `internal/*` package. Test files MAY import `internal/testfx/statefs`.
7. PR Boundary: no PR-D scope, no M3 work, no production callers, no spec edits. If you discover scope creep is unavoidable, STOP and write a proposal under `ai/proposals/`.
8. The repo carries a persistent local environment quirk where `kiox -- orun plan --changed --intent examples/intent.yaml` may fail on composition-cache resolution. Reproduces from Task 0001 onward on main. CI is authoritative; record the local result in the report and move on if it reproduces.
